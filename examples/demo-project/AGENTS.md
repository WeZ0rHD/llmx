## Workflow

Before starting work:
1. Read `project.llmx/memory/project-state.md`.
2. Read `project.llmx/memory/tasks.json` and pick the next open task.
3. Read the most recent entries of `project.llmx/memory/decisions.jsonl`.
4. Read `project.llmx/sessions/latest.md` for context from the last session.
5. Read `project.llmx/skills/index.json` and load any skill that matches your task.

During work:
- Do not duplicate existing architecture — check `project-state.md` first.
- Prefer adding a decision entry to `decisions.jsonl` over free-form prose.
- Use skills from `project.llmx/skills/` when they apply.

After work:
1. Update `project.llmx/sessions/latest.md` (via `llmx session save`).
2. Add decisions to `project.llmx/memory/decisions.jsonl`.
3. Update task statuses in `project.llmx/memory/tasks.json`.
4. Mention changed files in the session summary.
