# `docs/internal/` — author-only working notes

This directory contains **working notes, scratch ideas, and aspirational
backlogs** for the LLMX project. Files here are **not** part of the public
project narrative. They are not linked from the README, not advertised
in the spec, and not a commitment to ship anything.

## What goes here

- Long-horizon ideas (12+ months out)
- Strategy drafts and "what if we..." explorations
- Design rationales that didn't make it into the spec
- Internal RFCs in early draft state

## What does NOT go here

- The v0.x roadmap → `docs/roadmap.md`
- The product vision → `docs/vision.md`
- Anything a new contributor or user needs to read to use LLMX

## Files

- `horizon-2.0.md` — aspirational backlog for a v2.0 release
  (universal context protocol, browser ingest, sync SaaS).
  **Not a roadmap. Not a commitment. Read at your own risk.**

---

Rule of thumb: if a new user needs to read it to use the product,
it belongs in `docs/` at the top level. If only the author needs to
read it to remember what they were thinking, it belongs here.
